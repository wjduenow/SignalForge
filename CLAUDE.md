Three subcommands: `generate` (the full draft → prune → grade → diff pipeline against a single model), `lint` (config-only validator that loads `signalforge.yml`, the dbt manifest, and the warehouse profile without making any LLM or warehouse call), and `version` (prints `signalforge <__version__>`). DEC-021 sidecar-then-raise ordering for `GradeBelowThresholdError` (graduated from grade-layer's v0.2 reservation): `cmd_generate` writes the diff sidecar BEFORE raising the threshold error, so the operator's reproducibility artefact is always durable even on tier-2 failure. No traceback ever leaks (DEC-016): every `cmd_<name>` wraps its pipeline in a single `try / except Exception`, calls `format_error_to_stderr`, returns the mapped exit code; `_safe_excepthook` is installed via `sys.excepthook` for belt-and-braces unless `--verbose` is set (KeyboardInterrupt / SystemExit pass through unchanged). Stderr message shape is `ERROR: <message>` + optional `↳ Remediation:` footer for tier 1/3 and most tier 2; multi-violation tier 2 (drafter anchor-contract, lint multi-block) gets a header line plus `  - <violation>` bullets via `format_error_to_stderr` as the single sink — typed errors never override `__str__` for the bullet shape. Path canonicalisation flows through `canonicalise_user_path(raw, project_dir)` for every user-supplied path; failures re-raise as `CliPathError` so the CLI's catch surface stays homogeneous. Logger grep-gate extends to `src/signalforge/cli/` (6 dirs as of #9). One subprocess-gated smoke test (`pytest -m cli_subprocess`) catches `[project.scripts]` regressions that in-process `main(argv)` testing can't. - **#104 (ingest external schema.yml — library seam)** — `signalforge.ingest` subpackage: `read_schema(schema, model, *, project_dir=None) -> IngestResult` parses an externally-authored dbt `schema.yml` (hand-written, or from dbt-codegen / dbt Copilot / DinoAI / datapilot) into the typed `CandidateSchema` the prune engine already consumes — so SignalForge can prune *any* generator's tests, not just its own LLM drafts (extends Architectural Commitment #1). A **stage-0 reader** like `signalforge.manifest`: deterministic, no warehouse/LLM call, no audit writer, **no logging** (manifest-readers conventions). `schema` is disambiguated by type — `Path` is a file (canonicalised via `_common.path_safety`), `str` is raw YAML content. The four supported dbt test types map directly (`not_null`/`unique`/`accepted_values`/`relationships`); everything else (custom / namespaced / dbt-expectations) is **skip-and-recorded** into `IngestResult.skipped` as structured `SkippedTest` records (closed `SkipReason` literal: `unsupported-test-type` / `custom-or-generic-test` / `malformed-supported-test`), never failed loud (DEC-003). A test referencing a column absent from the `Model` **fails loud**, collecting every violation into one `IngestAnchorContractError` (DEC-002; re-implemented in-layer per DEC-007, NOT importing the draft validator, to avoid an LLM-typed error on a non-LLM path). Size-cap-before-`yaml.safe_load` (5 MB, DEC-005, mirrors diff's billion-laughs defence). Parser tolerates both `tests:`/`data_tests:` keys (unioned + deduped by `(type, column, sorted-args)`, DEC-008), inline + `arguments:`-nested args, interleaved config keys, and unwraps `ref()`/`source()` in `relationships.to` (DEC-009). `signalforge.ingest.errors` is the **11th per-stage `errors.py`**: five concretes registered in `_EXCEPTION_TO_EXIT_CODE` (3× tier 1, 2× tier 2), `IngestError` added to `_EXCEPTION_MAPPING_EXCLUDED_BASES` (frozenset-only, no fallback tier — concretes span tiers 1 and 2, like `DemoError`), scan-7 module count bumped 10→11. The operator-facing `signalforge prune-existing` CLI subcommand shipped in fast-follow [#105](https://github.com/wjduenow/SignalForge/issues/105) (see the #105 entry below). See `plans/super/104-ingest-external-tests.md` for the design + decisions, `docs/ingest-ops.md` for the operational reference, and `.claude/rules/ingest-layer.md` for the rules distilled from this ticket.

- **#105 (`signalforge prune-existing` CLI subcommand)** — `signalforge.cli.prune_existing` subpackage exposing the operator-facing `signalforge prune-existing <model> --schema <path>` subcommand: runs **ingest → prune → diff** with **no LLM call** (no draft, no grade) so an operator can point SignalForge at an existing dbt `schema.yml` (hand-written, dbt-codegen, dbt Copilot, …) and let the warehouse tell them which tests add no signal (extends Architectural Commitment #1 to any generator's tests). Flag set (DEC-002): positional `<model>` (bare-name / unique_id / file-path, resolved via the hoisted `signalforge.cli._helpers._resolve_model_by_key`), required `--schema`, `--project-dir` / `--manifest` / `--profiles-dir`, `--scope {sample,full}` + `--sample-strategy {oneshot,materialised}` (the prune knobs — `--mode` was **dropped** because `prune_tests` never consumes the `SafetyPolicy` on a no-LLM path), `--format {ansi,markdown,json}`, `--dry-run`, and the `--quiet`/`--verbose`/`--no-color` triad. **Read-only by design** (DEC-003): no `--write` (the `--schema` source is hand-authored); the rendered diff prints to stdout + a default-on `.signalforge/diff.json` sidecar (`--dry-run` suppresses it). The external schema.yml is fed to `render_diff(existing_schema=...)` so the unified diff shows what to *remove* from the operator's real file; `grading_report=None` → kept/kept-uncertain/dropped, never `flagged` (DEC-004). **No bespoke `CliPruneExisting*` error wrappers** (DEC-006): the five `IngestError` concretes were registered in `_EXCEPTION_TO_EXIT_CODE` by #104, so the handler's single `try/except Exception` boundary maps them directly. Skipped (unsupported) tests render as one stderr summary grouped by `SkipReason` + per-item detail under `--verbose` (DEC-007, control-chars scrubbed). 3-stage progress (`ingest`/`prune`/`diff`); `@pytest.mark.cli_subprocess` `--help` smoke + 5-surface parity test (`tests/cli/test_5_surface_parity_prune_existing.py`). See `plans/super/105-prune-existing-cli.md` for the design + decisions, `docs/cli-ops.md` § `signalforge prune-existing` for the operational reference, and `.claude/rules/cli-layer.md` / `.claude/rules/ingest-layer.md` for the rules.

## What SignalForge is

A CLI that drafts dbt `schema.yml`, tests, and docs with an LLM, then **prunes** the candidates against real warehouse data so only signal-bearing artifacts ship. The differentiator vs. dbt Copilot / dbt-codegen / DinoAI / datapilot is the prune step — competitors generate; SignalForge generates *and grades*.

## Architectural commitments (load-bearing — preserve when implementing)

These are stated in the README as design principles, not aspirations. New code should respect them:

1. **Signal over volume.** A candidate test that always passes on warehouse samples must be dropped, not shipped. Always-pass = no signal = worse than nothing because it consumes reviewer attention. Code paths that emit artifacts without running them through the prune step are a bug.
2. **Evaluation in the loop.** The grading layer reuses [clauditor](https://github.com/wjduenow/clauditor)'s LLM-as-judge methodology. Doc/artifact quality is scored against a configurable rubric — don't add ungraded artifact classes.
3. **Warehouse-agnostic by design.** Adapters plug in behind a thin sampling/profiling interface. v0.1 ships **BigQuery** only (chosen for query-bytes pricing on sampled reads + `INFORMATION_SCHEMA.JOBS` history). Snowflake/Postgres come in v0.2; Databricks/Redshift later. Don't bake BigQuery-isms into core — keep the adapter seam clean from day one.
4. **OSS-first, Core-friendly.** No dependency on dbt Cloud. Must run against any dbt-core project, locally or in CI.
5. **Explainable diffs.** Every kept/dropped artifact ships with a one-line "why." Don't add black-box code paths that drop or keep artifacts without recording the reason.

## Pipeline shape (per README)

```
model.sql + manifest + project ctx
  -> LLM drafts candidate artifacts
  -> run candidates against warehouse samples
  -> drop always-pass tests; drop tests that fail on known-clean data
  -> emit graded YAML + diff with per-artifact "why"
```

The "drop tests that fail on known-clean data" branch is as important as the always-pass branch — both directions of noise need pruning.
